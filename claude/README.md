# Claude Knowledge Base

Keep this tree small, clear, and easy to grow.

## Folders

- `principles/`: Timeless engineering ideas
- `review/`: How to evaluate code quality
- `standards/`: Concrete coding rules
- `workflows/`: Step-by-step playbooks
- `context/`: Environment and project notes

## Conventions

- One idea per file
- Use short headings
- Prefer checklists over long prose
- Delete stale guidance quickly
